# UI 规范与示例（语音故事拼图）

这是一套面向当前项目的轻社交、低压力体验的 UI 规范。目标是让用户在同屏共享场景中感到被倾听、易上手、无压力。

## 1. 设计原则
- 低压力优先：减少评判感，更多引导式提示。
- 单屏共享优先：关键状态需一眼可见（当前玩家、进度、倒计时）。
- 温和但有辨识度：色彩柔和，信息结构清晰。

## 2. 文字与排版
建议字体（可替换为设计稿中的字体）：
- 中文：思源黑体 / HarmonyOS Sans SC / 阿里巴巴普惠体
- 英文：Manrope / Source Sans 3

字号体系（页面为同屏/大屏）：
- H1 标题：20-24
- H2 模块标题：16-18
- 正文：13-15
- 辅助说明：11-12
- 数字强调（倒计时/完成度）：18-22

行高建议：正文 1.4-1.6，辅助说明 1.4

## 3. 色彩体系（可作为 token）
主色（品牌强调）：
- Primary: #4FD1C5
- Primary Dark: #3BAEA4

辅助强调：
- Accent Warm: #FFD45B
- Accent Coral: #FF7B7B
- Accent Purple: #6C63FF

中性灰：
- Text: #252237
- Subtext: #66607A
- Line: rgba(47, 42, 68, 0.12)

背景：
- Page BG: #F7FAFF
- Card BG: #FFFFFF

## 4. 空间系统
间距（8pt 基准）：
- XS: 4
- S: 8
- M: 12
- L: 16
- XL: 24
- 2XL: 32

圆角：
- 卡片/按钮：8
- 气泡/小标签：999

阴影：
- 卡片：0 8px 24px rgba(47, 42, 68, 0.10)
- 强调按钮：0 10px 24px rgba(255, 123, 123, 0.22)

## 5. 组件规范
### 5.1 语音按钮（主）
- 形态：圆角 8px 方形按钮
- 尺寸：42x42（主）、30x30（补充）
- 状态：
  - Idle：浅渐变底色 + 细描边
  - Recording：暖色渐变底色
  - Disabled：低对比浅色

### 5.2 提问卡
- 两种类型：画面描述式 / 感受描述式
- 标题 + 简短提示语
- 内含语音补充按钮与示例 chips

### 5.3 拼图区域
- 4x4 固定比例
- 中央舞台，保持一致尺寸
- 点击后选中高亮

### 5.4 进度区
- 当前玩家卡片 + 进度三格
- 倒计时采用高对比数字

### 5.5 完成弹窗
- 轻庆祝动画 + 最终图预览
- 两个按钮：继续查看 / 再玩一次

## 6. 动效规范
动效目标：增加反馈感，避免嘈杂。
- 入场：卡片淡入 + 轻微上浮
- 录音：按钮颜色变化 + 图标波形
- 生成中：加载文案 + 轻微渐变
- 完成：弹窗弹出 + 彩纸散开

## 7. 示例（文字版结构）

**页面结构示例**
- 顶部流程条
- 左侧：中央拼图舞台
- 右侧：当前玩家 + 语音生图区
- 右侧下方：同伴提问区
- 底部：风格选择 + 生成按钮

**按钮文案示例**
- 开始说话 / 停止并识别 / 识别中
- 生成图片 / 再玩一次 / 继续查看

**提示语示例**
- 说一件最近发生的小事就好
- 先补细节，再生成画面
- 画面里有什么？在哪里？远处能看到什么？
- 当时是什么感觉？像什么颜色？

## 8. 使用方式
- 把颜色、字号、间距做成 design tokens
- 组件按上述规范统一状态与尺寸
- 保持“低压力、被倾听、共同完成”的语气
